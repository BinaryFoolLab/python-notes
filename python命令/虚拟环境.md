# 虚拟环境

> 虚拟环境的核心价值是：让每个项目拥有独立、可控、可复现的运行环境

## 虚拟环境基本操作

> 创建完成后，应当先激活虚拟环境，再进行依赖安装、导出等操作

### 创建虚拟环境

> 一般建议创建在项目根目录中，`.venv` 为虚拟环境名称，可自定义

#### Linux/macOS

```shell
python3 -m venv ./.venv
```

#### Windows

```shell
python -m venv ./.venv
```

### 激活虚拟环境

#### Linux/macOS

```shell
source ./.venv/bin/activate
```

#### Windows

```shell
.\.venv\Scripts\activate
```

### 安装依赖包

```shell
pip install package_name
```

### 导出 / 导入依赖版本

> `requirements.txt` 通常用于项目环境复现，方便他人在相同环境下运行项目

#### 导出依赖

```shell
pip freeze > requirements.txt
```

#### 导入依赖

> 根据导出的依赖版本信息重新安装项目所需包

```shell
pip install -r requirements.txt
```

### 退出虚拟环境

```shell
deactivate
```
