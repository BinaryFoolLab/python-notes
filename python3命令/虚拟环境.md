# 虚拟环境

> 虚拟环境的核心价值是：让每个项目拥有独立、可控、可复现的运行环境

## 虚拟环境基本操作

### 创建虚拟环境

> 一般建议创建在项目根目录中，`.venv_name` 为虚拟环境名称，可自定义

```shell
python -m venv ./.venv_name
```

### 激活虚拟环境

#### Windows

```shell
.\.venv_name\Scripts\activate
```

#### macOS/Linux

```shell
source ./.venv_name/bin/activate
```

### 安装依赖包

> 需要先激活虚拟环境，再执行安装。安装后的依赖只会存在当前虚拟环境中，不会影响其他项目

```shell
pip install package_name
```

### 导出 / 导入依赖版本

> `requirements.txt` 通常用于项目环境复现，方便他人在相同环境下运行项目

#### 导出依赖

```shell
pip freeze > requirements.txt
```

#### 安装依赖

```shell
pip install -r requirements.txt
```

### 退出虚拟环境

```shell
deactivate
```
